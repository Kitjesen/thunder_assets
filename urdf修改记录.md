## 当前仍需注意的问题

### 1. 所有关节 effort / velocity 仍为 0

例如：

```xml
<limit lower="-0.41" upper="0.41" effort="0" velocity="0" />
```

这会影响 Gazebo、ros_control、MoveIt 或控制器使用。

### 2. 后腿 calf 关节限位仍不对称

当前：

```text
rr_calf_joint: -6.28 到 0
rl_calf_joint:  0 到 6.28
```

前腿为：

```text
fr_calf_joint:  0 到 3.14
fl_calf_joint: -3.14 到 0
```

如果四条腿结构一致，后腿 calf 关节限位建议进一步确认。

### 3. base_link 质心存在侧向偏移

当前：

```xml
<origin xyz="-0.00111 0.00718 0.07632" rpy="0 0 0" />
```

`Y = 0.00718 m`，约 `7.18 mm`。

### 4. ROS 包名为中文

当前包名和 mesh URI 使用：

```text
轮足狗机器人v3
```

例如：

```xml
package://轮足狗机器人v3/meshes/base_link.STL
```

ROS/catkin 工具链中更建议使用小写英文、数字、下划线形式的包名。

## 问题经验记录

### SolidWorks 装配左右未严格对称

曾经发现 URDF 中左右两侧部件的质心位置存在较大偏移。后续排查原因时，确认问题来自 SolidWorks 装配本身左右两侧没有做到严格对称。

影响表现：

- 左右腿对应 link 的 `<inertial><origin xyz="..." />` 不满足镜像关系。
- 同名对称部件的质心在 X、Y、Z 方向出现不应有的偏差。
- 导出的 URDF 中左右两侧关节/部件质量属性看起来不一致，容易误判为 URDF 编辑错误。

处理经验：

- 遇到左右质心不对称时，先回到 SolidWorks 检查装配约束、镜像关系和零件坐标系。
- 不要只在 URDF 中手动改数值掩盖问题；应优先修正 CAD 装配后重新导出质量属性。
- 对四足机器人这类左右/前后对称结构，应在导出后检查 `fr/fl`、`rr/rl`、`fr/rr`、`fl/rl` 的质心镜像关系。
